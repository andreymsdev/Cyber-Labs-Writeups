# Objetivos de Aprendizagem

- Entender por que gerenciar aplicações em servidores físicos individuais é ineficiente.
- Aprender como a virtualização resolve desafios de utilização de hardware e escalabilidade.
- Compreender os componentes de uma máquina virtual.
- Aprender como os containers otimizaram ainda mais a utilização de hardware para aplicações.

# Conclusão

- **Virtualização:** Permite que um único computador físico atue como múltiplos computadores separados.
- **Hypervisor:** O software "gerenciador" que cria e executa os computadores virtuais.
- **Máquina Virtual (VM):** Um computador virtual completo dentro do real, com seu próprio sistema operacional.
- **Container:** Uma "caixa" isolada e leve para uma aplicação que compartilha o mesmo sistema do hospedeiro.
- **Imagens de Container:** Uma receita/template pré-pacotado usado para criar containers.
- **Portas de Rede:** Pontos de entrada numerados específicos que os apps usam para se comunicar pela rede.
